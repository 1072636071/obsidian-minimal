## Bug 报告

在 [GitHub 仓库](https://github.com/kepano/obsidian-minimal)提交新问题。在提交 Bug 报告之前，请按照以下步骤操作：

1. 更新到最新版本的 Minimal 主题，更新你的插件，并重启应用
2. 确保在禁用插件和代码片段的情况下可以复现该 Bug
3. 确保使用默认 Obsidian 主题时*无法*复现该 Bug
4. 搜索[未解决的问题](https://github.com/kepano/obsidian-minimal/issues)，确保该 Bug 尚未被报告
5. 将你的调试信息粘贴到问题报告中。可以在 Obsidian 中使用 `CTRL/CMD-P`，输入"Show debug info"，然后点击"Copy to clipboard"获取

## 功能请求

在 [GitHub 仓库](https://github.com/kepano/obsidian-minimal)提交新请求。在请求功能之前，搜索[未解决的问题](https://github.com/kepano/obsidian-minimal/issues)，确保该功能尚未被请求。

只提交 Minimal 能够处理的功能请求。对于一般的 Obsidian 功能请求，请使用 [Obsidian 论坛](https://forum.obsidian.md/)。对于插件的改进，请前往相应插件的 GitHub 仓库查看说明。

## 与社区 CSS 代码片段的兼容性

Obsidian 让你可以轻松创建和加载自己的 [[CSS 代码片段]]。Obsidian 社区成员提供了无数的代码片段，但并非所有片段都能保证与 Minimal 兼容。

使用社区代码片段需自行承担风险。Minimal 未针对社区代码片段进行兼容性测试。如果遇到代码片段的问题，请联系代码片段的作者。
